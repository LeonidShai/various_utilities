# Plan for Golang interview

## Gorutines

### 1 Что такое горутина?
Это блок кода, который работает асинхронно с какими-то другими блоками кода.
- Управляется рантаймом языка (Более высокоуровневая абстракция, поэтому не зависит от системы)
- легковесный (небольшой стэк ~2 Kb)

### 2 Может ли стэк горутины расти?
Да может. Максимум 1 ГБ для 64-разрядной архитектуры, 250 МБ для 32-разрядной архитектуры.

### 3 Что будет если размер горутины превысит допустимы максимум?
Если размер стэка горутины превышен (к примеру запустили бесконечную рекурсию), то приложение упадет с fatal error. stack overflow

### 4 Какое максимально кол-во горутин может быть? а одновременно работающих?
максимальное ограничено только оперативой. Работает по количеству логических ядер процессора.

### 5 Что такое планировщик?
(Кооперативный + вытесняющий)
M - машина, поток операционной системы, которым она управляет;
P - контекст планирования, необходимый для возможности переброски очередей планирования между потоками (M);
G - горутина, прежде всего там содержатся: стек, указатель команд, канал для планирования горутин.
Локальная очередь (LRQ — Local Run Queue) — у каждого P своя собственная очередь.
Глобальная очередь (GRQ — Global Run Queue) — общая очередь для всех P (защищена мьютексом). Сюда попадают горутины, созданные в одном P, но еще не назначенные.

1. Создание: Вы пишете go f(). Рантайм создает новую горутину (G) с небольшим стеком (2 КБ) и помещает ее в локальную очередь текущего P (если там есть место). Если локальная очередь заполнена — в глобальную.
2. Запуск: M (поток), связанный с P, берет следующую горутину из головы своей локальной очереди и начинает ее выполнять.
3. Переключение (Кооперативное): Горутина выполняется до тех пор, пока не наступит событие для переключения. (Блокировки канала/мьютексы. системные вызовы)
Если у какого-то P (процессора) закончились задачи в своей локальной очереди, он не простаивает. Он начинает воровать работу у соседей.

### 6 Какие есть способы остановить все горутины в приложении?
- завершение main функции и main горутины;
- прослушивание всеми горутинами channel, при закрытии channel отправляется значение по умолчанию всем слушателям, при получении сигнала все горутины делают return;
- завязать все горутины на переданный в них context.

## Synchronization of threads

### 1 Какие есть примитивы синхронизации?
- wait and error group;
- mutex;
- atomic;
- sync map;
- channel.

### 2 В чем отличие waitGroup от errorGroup?
errgroup умеет работать с ошибками и отменять все горутины при сбое одной из них.

### 3 Что такое мьютекс? И какие бывают? на чем основан? Для чего нужны?
sync.Mutex - блокирует кусок кода как на запись, так и на чтение;
sync.RWMutex - позволяет блокировать кусок кода только на запись.
Нужны для избежания гонок данных.
```
type Mutex struct {
    state int32  // 4 байта: хранит состояние (флаги и счетчик ожидающих)
    sema  uint32 // 4 байта: семафор для блокировки/разбуживания горутин
}
```
Быстрый путь: Атомарная операция CompareAndSwap.
Медленный путь:
- Активное ожидание (проверяет в коротком цикле не освободился ли ресурс)
- Ожидание (ждет сигнала, что освободился мьютекс)
Режим голодания: когда горутина долго спит в ожидании, при освобождении происходит прямая предача.

### 4 что если два раза подряд вызвать Lock?
будет дедлок

### 5 что такое atomic?
это механизм, который позволяет выполнять простые действия над переменными как единое, неделимое целое прямо на уровне процессора, без использования блокировок (таких как мьютексы)

### 6 В чем отличие буферезированного канала от небуферезированного?
Небуферизированный канал — синхронный. Отправитель блокируется до тех пор, пока получатель не будет готов забрать значение.
Буферизированный канал — асинхронный (до заполнения буфера). Отправитель блокируется только тогда, когда буфер полностью заполнен.

Отправка/чтение в/из небуферизированный канал без получателя приводит к дедлоку.
Запись в закрытый канал приведет к панике.

### 7 Как можно читать данные из канала?
- присвоение в переменную
- for range
- select case

### 8 Как сделать select неблокирующим?
добавить default

### 9 Какой порядок исполнения операций case в select?
Первым выполнится тот case в select, который будет готов. При одновременной отправке данных в каналы, прослушиваемые в select порядок операций не гарантирован.

### 10 Что такое context?
Стандартный механизм для передачи сигналов отмены, дедлайнов (таймаутов) и ключей/значений через границы API и между горутинами.
У контекста есть родитель, и если отменить родителя — отменятся все его дети.
Отмена контекста через cancel().


## OOP and methods

### 1 Как реализовано ООП в Go?
- Инкапсуляция через приватные/публичные имена переменных/методов/функций в пакете
- Наследования нет, есть встраивание.
- Полиморфизм через интерфейсы.

### 2 Как сделать свои методы для стороннего пакета?
Для этого нам нужно будет создать свою структуру, внутри в неё встраить структуру стороннего пакета, и к этой нашей структуре уже прикручивать требуемые методы. Просто "навесить сверху" функции на сторонний пакет мы не можем.

### 3 Что такое интерфейс?
Смотри рядом

### 4 Когда интерфейс будет равен nil?
Интерфейс будет равен nil тогда и только тогда, когда оба его поля равны nil.

## Memory

### 1 как устроена память в go?
- стек
- куча

### 2 Побег памяти?
Если память "убегает": Вы возвращаете указатель на переменную из функции, или передаете её в канал, или присваиваете глобальной переменной. Компилятор выделяет её в куче.

### 3 Как происходит передача параметров в функцию?
по значению

### 4 Есть ли особенности поведения при передаче map и slice в функцию?
Структуры slice и map (уточнение: в случае map копируется не сама структура, а указатель структуру hmap) копируются, однако в самих структурах содержатся ссылки на области памяти, благодаря которым создается эффект передачи по ссылке.

### 5 Сборщик мусора
смотрив соседнем

## Data structures

### 1 Что представляют из себя строки в go?
массив байт

### 2 Как определить количество символов для строки? Какие есть нюансы при итерации по строке?
Исходя из того же знания, что строка это массив байт, взяв базовую функцию len() от строки мы получим количество байт. Похожее поведение будет при итерации по строке - итерация по байтам. Тогда как в зависимости от кодировки, символ в строке может занимать не один байт.

Для того, чтобы работать именно с символами, необходимо преобразовать строку в тип []rune.

### 3 Что такое слайс и чем он отличается от массива?
```
type slice struct {
	array unsafe.Pointer
	len   int
	cap   int
}
```
Если на основании слайса one выделить подслайс two, а затем увеличим слайс one (и его вместимость будет превышена) — то one и two будут уже ссылаться на разные участки памяти!
